Below is a compile-safer .NET MAUI project version.
I’ve simplified a few things to reduce XAML/parser risk:
•	no custom complex routing framework
•	no dynamic bindable custom shell properties beyond what is needed
•	all page switching is done in DashboardShellPage.xaml.cs
•	sidebar uses normal event handlers instead of fragile CollectionView selection bindings
•	top header binds directly to the shell view model
•	shared components are minimal
•	pages are simple, strongly aligned to their view models
This version is meant to be much closer to:
paste → fix namespaces if needed → build

11. Compile Notes
Safer decisions made here
•	SidebarView uses button click events, not CollectionView selection logic
•	DashboardShellPage swaps content using page.Content
•	shared controls only use simple bindable properties
•	no command parameters needed for shell navigation
•	no complicated custom controls for tables
If anything still fails
The most likely causes are:
1.	namespace mismatch between folder and x:Class
2.	a missing using Microsoft.Extensions.DependencyInjection; for GetRequiredService
If needed, add this at the top of DashboardShellPage.xaml.cs:
csharp
using Microsoft.Extensions.DependencyInjection;
